To set up a Django project with a virtual environment, create and activate a Python venv, install Django inside it, then
generate the project structure.[1][3]

## Prerequisites

Ensure Python 3 and pip are installed. Optionally install `virtualenv` via `pip install virtualenv`, though `venv` is
built-in.[2][1]

## Step-by-Step Setup

- Create a project directory: `mkdir my_django_project && cd my_django_project`.[1]
- Generate virtual environment: `python -m venv venv` (replace `env` with your preferred name).[3][1]
- Activate it:
  - Windows: `env\Scripts\activate`[3][1]
  - macOS/Linux: `source venv/bin/activate`[2][3] Your prompt will show `(env)` when active.
- Install Django: `pip install django`.[1]
- Create project: `django-admin startproject myproject .` (use `.` for flat structure).[1]
- Run server: `python manage.py runserver` to test.[3]

## Best Practices

Add `env/` to `.gitignore`. Reactivate the venv each session. Use `pip freeze > requirements.txt` for dependencies and
`pip install -r requirements.txt` to recreate. Deactivate with `deactivate` when done.[4][5][3]
