## Overview

Use a `.env` file in Django to securely store sensitive data like secret keys and database credentials as environment
variables, loaded via packages like `python-dotenv` or `django-environ`.[1][3]

## Installation

Install `python-dotenv` with `pip install python-dotenv` for basic loading, or `django-environ` for advanced features like
type casting.[2][1]

## Setup Steps

- Create a `.env` file at your project root (same level as `manage.py`), using `KEY=VALUE` format without quotes for strings:
  ```
  SECRET_KEY=your-secret-key
  DEBUG=True
  DATABASE_URL=postgres://user:pass@localhost/dbname
  ```
  Add `.env` to `.gitignore` to avoid committing secrets.[3][6][1]
- In `settings.py`, load variables early:
  ```python
  import os
  from dotenv import load_dotenv
  load_dotenv()  # Loads .env into os.environ
  ```
  Then access with `os.getenv('SECRET_KEY')` or booleans like `os.getenv('DEBUG') == 'True'`. For `django-environ`:
  ```python
  import environ
  env = environ.Env(DEBUG=(bool, False))
  environ.Env.read_env(BASE_DIR / '.env')
  SECRET_KEY = env('SECRET_KEY')
  ```

## Accessing Variables

Replace hardcoded values in `settings.py`:

- `SECRET_KEY = os.getenv('SECRET_KEY')`[3]
- Database config: `'NAME': os.getenv('DB_NAME')`[4] Use defaults like `os.getenv('KEY', 'default')` for safety.[6]

## Production Notes

In production, set variables via server environment (e.g., DigitalOcean dashboard) instead of `.env` files for security;
avoid checking `.env` into Git.[8][10]
