To add Sass and JavaScript scripts to a Django app, follow these key steps:

For JavaScript:

- Place your JavaScript files in a static directory within your app or project (e.g., myapp/static/js/app.js).
- In your Django template, load static files with `{% load static %}` at the top.
- Include the JavaScript file in the HTML by referencing it via the static template tag, e.g.,
  `<script src="{% static 'js/app.js' %}"></script>`. This will ensure Django serves the JS file correctly with static files
  handling.[1][3][5]

For Sass (SCSS):

- Install packages like `django-compressor` and `django-libsass` to handle SCSS compilation within Django.
- Add `compressor` to your `INSTALLED_APPS` and configure `STATICFILES_FINDERS` and `COMPRESS_PRECOMPILERS` in settings, for
  example:
  ```
  INSTALLED_APPS += ['compressor']
  STATICFILES_FINDERS += ['compressor.finders.CompressorFinder']
  COMPRESS_PRECOMPILERS = (
    ('text/x-scss', 'django_libsass.SassCompiler'),
  )
  ```
- Place your `.scss` files inside the static directory, then in templates load compressor tags `{% load compress %}` and use
  a compress block to include SCSS files, e.g.,
  ```
  {% compress css %}
  <link rel="stylesheet" href="{% static 'styles/main.scss' %}" type="text/x-scss">
  {% endcompress %}
  ```
- When running the Django server, the SCSS will be compiled automatically to CSS and served.[2][4][6]

This setup allows you to write Sass stylesheets and JavaScript scripts in your Django project with automatic compilation and
static file serving handled by Django. You just need to restart the server after making these changes for them to take
effect.
