Python does not have a built-in operator exactly like JavaScriptâ€™s optional chaining (`?.`), but you can achieve similar
behavior using:

- The `dict.get()` method for dictionaries, where you can provide default values to avoid exceptions on missing keys, e.g.
  `data.get("a", {}).get("b")`.
- Custom utility functions or classes that safely traverse nested attributes or keys, returning `None` (or another default)
  instead of raising exceptions.
- The `getattr()` function with a default value to safely access object attributes, e.g. `getattr(obj, "attr", None)`.

Example function mimicking optional chaining for mixed dict/object access:

```python
def optional_chain(root, *keys):
    result = root
    for key in keys:
        if isinstance(result, dict):
            result = result.get(key, None)
        else:
            result = getattr(result, key, None)
        if result is None:
            break
    return result
```

Usage:

```python
obj = {'a': {'b': {'c': 1}}}
print(optional_chain(obj, 'a', 'b', 'c'))  # outputs 1
print(optional_chain(obj, 'a', 'x', 'c'))  # outputs None
```

This technique safely accesses nested values without throwing exceptions if any part is missing, similar in spirit to JS
optional chaining.[1][3][4]

In summary, Python lacks an exact `?.` operator but provides multiple idiomatic ways to achieve similar safe navigation
through nested data.
