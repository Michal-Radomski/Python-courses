To use a .env file in Python scripts to hide API keys and other secrets (similar to Node.js), you can use the `python-dotenv`
package.

### Steps to use .env in Python:

1. **Create a .env file** in your project directory with your secrets, e.g.:

   ```
   API_KEY=your_api_key_here
   SECRET_KEY=your_secret_key_here
   ```

2. **Install the python-dotenv package**:

   ```
   pip install python-dotenv
   ```

3. **Load the .env file in your Python script** using:

   ```python
   from dotenv import load_dotenv
   import os

   load_dotenv()  # Loads the .env file contents into environment variables

   api_key = os.getenv("API_KEY")
   secret_key = os.getenv("SECRET_KEY")

   print(f"API Key: {api_key}")
   print(f"Secret Key: {secret_key}")
   ```

This works by loading the variables from your .env file into the process environment variables, allowing you to access them
via `os.getenv()`.[1][2][5]

This approach keeps your sensitive data out of the source code and prevents them from being committed to GitHub when you
include `.env` in `.gitignore`.
